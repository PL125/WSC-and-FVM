Ⅰ．Getting started

将bin文件夹内的3个文件（WSC.g1a, FVM.g1a, WSCLIB）复制到计算器内。为了编辑源代码，你可能还需要用9860 的EDITv1.51 ，当然你也可以在电脑上编辑源代码，它是纯文本格式的。

安装完成后，主菜单应如下：

打开EDIT，在Storage　Mem内（注意不是Main Mem！）中建立新文件，

输入文本：

void main( )

{

    printf( “Hello World!\n” );

}

(P.S. edit可以调字体大小（shift—set up进入设置界面），用小字在一个屏幕上可以看到更多代码，推荐使用小字，在optn里的font size里设置，另外word wrap也推荐选off，推荐设置如下图)

输入完成后如下：

保持文件，退出，进入WSC，按F1（compile）编译源代码。

如果没有错误，屏幕会输出Done，如有错误，可返回EDIT修改。完成后任意键退出，进入FVM，按F1（run）执行字节码。屏幕就会输出hello world。执行完毕后任意键退出。

Ⅱ．Features

1. 操作符

+ - * / % += -= *= /= < <= > >= == != && || ! != &

注意最后一个‘&’是取地址而不是按位与

2. 数据类型

int char float 任意维数的数组 指向基本类型的指针（不能指向数组）

字符串常量，16进制常量

3. 语句

if else while for return break

4. 注释

C风格的 /*...........*/ 和 C++风格的 //...........

5. 函数

支持递归, 详细库函数列表见 library.html

6. 预处理器

简单的宏替换 #define （不能带参数）， #include（不是真正的文件包含，内部使用）

Ⅲ．Error reporting

wsc的错误报告还不是很完善，计算器过小的显示屏也不方便显示所有的错误信息。

当有错误被检测到时，wsc会在屏幕上显示错误信息，如图：

这时按menu键可以退出修改，其他键显示下一条错误信息。

注意WSC没有提示警告！

任何指针、数组在它眼里都是相同的，可以任意赋值。

int、char、float之间的赋值，也不会产生任何警告信息。

一个返回类型非空的函数没有return语句也不会产生警告信息。

Ⅳ．Differences

以下是WSC与标准C的一些不同之处

1. 不需要函数声明

只要在文件中定义了函数，不管位置，都可以随处调用。

2. 只有两个作用域

整个程序只有两个作用域，全局作用域和函数内的局部作用域。

void main()

{

int a;

{

    int a;

}

}

例子程序会收到一个错误消息,“Variable 'a' redefinition”

3. int * a, b；

在WSC中a和b都会是指向int的指针。（所以建议写成int*  a， b； 而不是int  *a，b ；）。

4. int *p； p++；

ANSI C中实际p的值会加4（即sizeof（int）），但是WSC只会让p加1。

5. char s[50] = “asdfjkl”；

这种初始化方式暂未被支持，例句是错误的。

6. int a[2][2] = { 1, 2, 3, 4 }；

这种初始化方式暂未被支持，例句是错误的。多维数组的初始化必须正确分组，如

int a[2][2] = { {1, 2}, {3, 4} }；分组正确后少一些元素也正确，如

int a[2][2] = { {1}, {3} }；

Ⅴ．Notice & Tips

1. int a[10]；printf( “%d”, a )；

代码编写者的意图是获取a的值，但是如上文所说，WSC对所有指针都看做一个类型，printf是重载的库函数，第二个参数是指针的函数只有printf( char *, char * )，所以编译器做了一个错误的选择，把a看成了char *。

一种正确的作法是int a[10]; int t = a; printf( “%d”, t );

2. char ch；scanf( “%c”, &ch )；

出于某些原因，这种写法会出现问题，推荐使用getchar（）

3. FVM会自动初始化随机数种子

不需要程序编写者调用srand（）

4. 输入大写字母

在输入界面按F6可以切换大小写

5. 设置多键强退

#progma FORCE_BREAK

在源代码任意位置添加上面的语句，即可在执行程序的时候同时按住F1 + F2 + F6来强制退出程序

6. 在IO界面使用大字体

#progma BIG_FONT

在源代码任意位置添加上面的语句即可

7. FVM程序的大小限制

FVM程序最大可达到10kb（不是源代码，是.f文件），如果你的程序超出这个范围，请使用exefvm函数

8. FVM的执行环境

默认内存（此处指静态区和栈区）8kb，可以通过以下语句重新定义

#progma RAM_SIZE 10240 // 设置为10240kb = 10kb

堆区的大小取决于计算器

如果你需要exefvm其他程序，建议把被调用的程序的RAM_SIZE设置的小一点
