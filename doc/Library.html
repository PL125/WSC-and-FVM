WSC Functions

使用以下函数或定义需要包含 fxlib.h

图形函数操作的都是显存，需要使用putdisp()把显存里的内容输出到屏幕

点的横坐标范围是0~127，纵坐标范围是0~63

 

 

void allclr( void )

清除全部显存

 

void areaclr( int x1, int y1, int x2, int y2 )

清除指定矩形区域的显存，（x1,y1）是矩形左上角，（x2，y2）是矩形右下角

 

void arearev( int x1, int y1, int x2, int y2 )

对指定区域矩形区域进行反转颜色，（x1,y1）是矩形左上角，（x2，y2）是矩形右下角

 

void putdisp( void )

将显存输出的屏幕上

 

void setpoint( int x, int y, char kind )

对点（x，y）进行设置，kind值决定颜色， 1是黑色，0是白色

 

int getpoint( int x, ing y )

返回点（x，y）的眼色，1是黑色，0是白色

 

void drawline( int x1, int y1, int x2, int y2 )

画一条（x1，y1）到（x2，y2）的黑线

 

void clearline( int x1, int y1, int x2, int y2 )

画一条（x1，y1）到（x2，y2）的白线

 

void drawcircle( int x, int y, int r )

在（x，y）画一个半径r的空心圆

 

void fillcircle( int x, int y, int r )

在（x，y）画一个半径r的实心圆

 

void drawbox( int x1, int y1, int x2, int y2 )

画一个空心矩形框，（x1,y1）是矩形左上角，（x2，y2）是矩形右下角

 

void fillbox( int x1, int y1, int x2, int y2 )

画一个实心矩形框，（x1,y1）是矩形左上角，（x2，y2）是矩形右下角

 

void locate( int x, int y )

设置输出光标的位置为（x，y） 这里x的范围是1~21，y的范围是1~8

 

void print( char *str )

在光标位置输出字符串

 

void printxy( int x, int y, char *str, int type )

在（x,y）位置输出字符串，type值等于1时反色，等于0时正常

 

void printmini( int x, int y, char *str, int type )

在（x,y）位置输出字符串（小字体），type的值见下表

MINI_OVER

覆盖

MINI_OR

OR

MINI_REV

反色

MINI_REVOR

反色 + OR

 

void savedisp( char num )

把显存保持到一个标号为num的系统区域，num的值只能是SAVEPAGE1, SAVEPAGE2, SAVEPAGE3

 

void restdisp( char num )

把之前用savedisp保持的显存重新设置到显存区

 

void popupwin( int n )

弹出一个n行的窗口，n的范围是1~5

 

int openfile( const char *name, int mode )

name是文件名（查看文件名格式）

mode决定打开方式，只能是以下的值

OPEN_R

读文件

OPEN_W

写文件

OPEN_RW

读写文件

如果打开成功，会返回一个特殊的文件句柄（handle）,它是一个大于等于0的值

如果打开失败，会返回一个错误代码，它是负数

 

int readfile( int handle, void *buf, int size, int readpos )

handle是openfile的返回值

readpos是开始读取的位置，如果它等于-1，那么将在上次读取结束的位置开始

其他参数和返回值与fopen类似

 

int writefile( int handle, void *buf, int size )

handle是openfile的返回值，其他与fwrite类似,

 

int seekfile( int handle, int pos )

handle是openfile的返回值，pos指定了读写文件的位置

成功返回0，失败返回负数

 

int closefile( int handle )

关闭文件

 

int getfree( int type, int *freebytes )

读取剩余可用内存。

type只能是下列值

MAIN_FREE

main memory

STO_FREE

storage memory

SD_FREE

sd card

设备剩余空间的值会存储在freebytes指向的int值中

成功返回0，失败返回负数

 

int getsize( int handle )

handle是openfile的返回值

返回文件大小

 

int createfile( const char *name, int size )

创建一个名字为name，大小为size的文件（查看文件名格式）

成功返回0，失败返回负数

 

int createdir( const char *name )

创建一个名字为name的目录（查看文件名格式）

成功返回0，失败返回负数

 

int deletefile( const char *name )

删除文件（查看文件名格式）

成功返回0，失败返回负数

 

int deletedir( const char *name )

删除目录（查看文件名格式）

成功返回0，失败返回负数

 

 

以下三个函数需要配合使用，例子见example\find.c

int findfirst( char *pathname, int *handle, char *foundname, int *fileinfo )

pathname是寻找路径（查看文件名格式），可以使用通配符＂*＂,如＂\\\\fls0\\＊.txt＂、＂\\\\fls0\\＊＂

文件寻找句柄将保持在handle指向的int里（用于findnext和findclose）

foundname将保存找到文件的名字

fileinfo保持找到文件的信息，它应该是一个大小为6的int型数组（int fileinfo[6]），信息存储位置见下表

索引

信息

0

文件id

1

文件类型

2

文件大小

3

数据大小（除去文件头）

4

property（the file has not been completed, except when property is 0.）

5

地址（请不要直接访问这个地址）

文件类型见下表

值

类型

0

目录

1

文件

2

Add-In

3

eActivity

4

Language

5

Bitmap

6

Main Mem数据

7

临时数据

8

. （当前目录）

9

..（父目录）

10

卷标

成功返回0，失败返回负数

 

int findnext( int handle, char *foundname, int *fileinfo )

handle是findfirst的第二个参数指向的对象

foundname和fileinfo参考findfirst

成功返回0，失败返回负数

 

int findclose( int handle )

handle是findfirst的第二个参数指向的对象

成功返回0，失败返回负数

 

int getkey( int *keycode )

获取一个按键，按键编码保存在keycode指向的int值中

如果字符键被按下，返回1。如果控制键被按下，返回0。

如果按键队列为空，它会等待，直到一个按键被按下（查看按键代码）

 

int waitkey( void )

获取一个按键，并返回按键编码，是getkey的一种简化使用

如果按键队列为空，它会等待，直到一个按键被按下（查看按键代码）

 

int iskeydown( int keycode )

检测keycode是否被按下，如果按下就返回1，否则返回0

如果按键队列为空，这个函数不会等待。

 

void sleep( int millsecond )

延时millsecond毫秒

 

void cpuspeed( int multiple )

设定cpu时钟的速度，multiple是默认速度的倍数，取值范围是1~4

（4即4倍超频）

 

void resetcalc( void )

和按计算器背后的reset键效果相同

 

int readrtc( int type )

读取real time clock（实时时钟）,type只能是以下的值

RTC_YERA

年

RTC_MONTH

月

RTC_DAY

日

RTC_HOUR

小时

RTC_MINUTE

分钟

RTC_SECOND

秒

RTC_DAYOFWEEK

星期

 

void setrtc( int type, int value )

设置real time clock（实时时钟）,type的值见上表

关于rtc读写的例子见 example\CLOCK.c

 

int bitand( int a, int b )

按位与，即 a & b

 

int bitor( int a, int b )

按位或，即 a | b

 

int bitxor( int a, int b )

按位异或，即 a ^ b

 

int bitnot( int a )

按位取反，即 ~ b

 

int shiftl( int a, int b )

向左位移，即 a << b

 

int shiftr( int a, int b )

向右位移，即 a >> b

 

int exefvm( char *filename, char *msg )

执行另一个fvm程序，msg用来传递信息（查看文件名格式）

返回值就是程序main的返回值

 

char *getfvmmsg( void )

获取exefvm传递过来的参数，如果是直接由FVM执行的，则返回NULL

关于信息传递的例子见example\msgpass\

 

 

 

 

 

 

 

文件名格式

举几个例子来说明文件名格式：

Storage Mem中根目录下的2.txt的名字是＂\\\\fls0\\2.txt＂

Storage Mem中目录book下的2.txt的名字是＂\\\\fls0\\book\\2.txt＂

sd卡中根目录下的2.txt的名字是 ＂\\\\crd0\\2.txt＂

sd卡中目录music的名字是＂\\\\crd0\\book＂

现在还不能处理Main Mem中的文件

注意文件名是大小写敏感的！

 

关于文件操作的例子见 example\FILE.c
